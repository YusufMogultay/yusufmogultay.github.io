Aim

The project name is Library Automation

Fast and secure presentation of records and book tracking for any library by transferring them to a computer environment.


Target Group

Municipal libraries, All University libraries, Private or Public High Schools, Institutions or organizations with a library.

justifications

• Lost books or books not delivered within a certain date.
• There are many books and it is difficult to keep a record.
• Non-regular tracking of the book delivery time.
• Too much effort and waste of time.
• People who are not registered in the system try to borrow books.
• Understanding whether the searched book is available or not.

returns

• It can be determined who has the last lost book or books.
• Thousands, maybe millions of books can be save into a database, and all information can be easily searched.
• The information of existing books can be updated.
• Can delete an existing book from the system.
• Can add new books to the system.
• Keeping the information about which book has been lent to whom for how many days, accurate and without doubt, using features such as book lending.
• View saved contacts in the system.
• Saving time and energy.

possible risks of use

Data loss may occur because the database doesn't have the option to take backups in automation. Because Excel also doesn't have the ability to transfer ready-made data to automation, if there is a lot of data, you can have to enter it all manually. The system gives a yes or no message before irreversible proccess like deleting, adding, updating, etc. The system prevents unwanted changes in this way.
